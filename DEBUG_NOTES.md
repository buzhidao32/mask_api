# 面具分数查询功能调试记录

## 最终方案

词库使用 `$替换$` 提取查询词，调用服务器 API 返回文本：

```
^@丹凤 面具.+$
$回调 queryMask$

^@丹凤 称号.+$
$回调 queryMask$

^@丹凤 (.{2,10})$
$回调 queryMask$

[内部]queryMask
Q:$替换 %参数0% @丹凤  $
E:$URLEncoder %Q%$
$回复 %MsgId%$
$访问 http://服务器IP:8080/api/score/text?name=%E%$
$发送$
```

## 踩坑记录

### 1. `%括号1%` 不生效
最初想用正则捕获组 `(.+)` 配合 `%括号1%` 提取查询词，但 `%括号1%` 始终返回空。

```
# 这样写 %括号1% 为空
^@丹凤 面具(.+)$
$回调 queryMask$

[内部]queryMask
查询词：%括号1%  # 始终为空
```

### 2. `$删除字符$` 在内部函数中不执行
尝试用 `$删除字符$` 从文本中删除前缀，但在 `[内部]` 函数中该函数不执行，直接输出了函数原文。

```
# 不生效
[内部]queryMask
Q:$删除字符 %Text% @丹凤 面具  # 输出函数原文而非结果
```

### 3. `$正则提取$` 不是标准 Sec 函数
最初使用的 `$正则提取$` 在 Sec 变量大全中不存在。

### 4. 服务器端 Python http.server 不支持中文 URL
直接用中文参数访问 API 会返回 400 错误，必须先用 `$URLEncoder$` 编码。

```
# 不生效
$访问 http://服务器/api/score/text?name=面具嫦娥$

# 生效
E:$URLEncoder 面具嫦娥$
$访问 http://服务器/api/score/text?name=%E%$
```

### 5. `^@.*[ 　]+面具.+$` 模式不触发
使用通配符 `.*` 匹配机器人名字的模式不生效，必须写死 `@丹凤`。

```
# 不生效
^@.*[ 　]+面具.+$

# 生效
^@丹凤 面具.+$
```

## 关键发现

1. **`%参数0%`** 包含完整的匹配文本（如 `@丹凤 面具嫦娥`）
2. **`$替换$`** 可以在 `[内部]` 函数中正常执行，用于提取查询词
3. **`$URLEncoder$`** 必须对中文参数编码，否则服务器无法解析
4. **词库触发规则** 必须写死机器人名字，不能用通配符
5. **`$回调$`** 调用 `[内部]` 函数时，`%参数0%` 继承自触发规则的匹配结果

## 服务器部署

服务器端口：8080
监听地址：0.0.0.0（必须，否则外部无法访问）

```bash
# 检查服务器状态
systemctl status mask-api

# 测试 API
curl "http://127.0.0.1:8080/api/score/text?name=%E9%9D%A2%E5%85%B7%E5%AB%A6%E5%A8%A5"
```
